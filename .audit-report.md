### 1. 之前的 HIGH 修复：「```」替换已实现但不完备

`stream.go:720` 将 preview 中的 ` ``` ` 替换为 `` `\u200b`` ``，防止三连反引号提前关闭外层代码块。**这部分正确**。

但 **遗漏了 MarkdownV2 在 code block 内的其他转义要求**。对照 `telegram_format.go:241-242`（已有的 `formatMessage` 函数）的标准做法：

```go
// telegram_format.go:241-242 — 正确做法
body = strings.ReplaceAll(body, `\`, `\\`)
body = strings.ReplaceAll(body, "`", "\\`")
```

当前代码只做了：
```go
// stream.go:720 — 不完整
safePreview := strings.ReplaceAll(preview, "```", "`\u200b``")
```

**缺失项：**

| 字符 | 需要转义为 | 风险 |
|------|-----------|------|
| `\` | `\\` | 末尾 `\` 可转义 closing fence 前的换行符，导致代码块无法关闭 |
| `` ` `` | `` \` `` | 单反引号在 MDV2 code block 内须转义 |
| ```` (4+ 反引号) | 应拦截 | CommonMark 下 ```` 即可关闭 ```-fenced block |

**威胁场景：** 攻击者通过 prompt injection 使 AI 生成带恶意 `preview` 的 tool call → 突破代码块 → 在审批消息中注入任意 MarkdownV2 → 改变用户看到的审批内容（社会工程）或导致消息发送失败（审批流程 DoS）。

**修复方向：** 将 line 720 替换为：
```go
safePreview := strings.ReplaceAll(preview, `\`, `\\`)
safePreview = strings.ReplaceAll(safePreview, "`", "\\`")
```
或者复用一个统一的内 code block 转义函数。
